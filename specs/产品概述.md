# 产品概述 (PayForMe)

## 1. 核心定位
PayForMe 是一个面向商户团队的综合商品推广与分销对账系统。
其核心价值在于通过严格的四级组织架构权限（首席工程师 - 经理 - 主管 - 员工），实现商品的统一分发、订单的全自动归集（Webhook 回调）、员工销售业绩的独立核算以及提现流程的线上化审批。

## 2. 真实系统能力 (基于代码现状)

### 2.1 账号与权限体系 (Role-Based Access Control)
系统在 `server.ts` 和前端路由中强约束了以下四个角色的边界：
- **首席工程师 (Chief Engineer)**
  - 全局最高权限（无视商户隔离）。
  - 可见所有用户树状图，可直接在节点上增加、删除、修改任何员工的收益比例。
  - 提供 `数据库管理` 界面，支持直接在前端执行增删改查 SQL。
  - 提供 `系统管理` 界面，支持增删改查 `system_configs` 全局变量。
- **经理 (Manager / Admin)**
  - 团队最高管理者。
  - 只能看到并管理自己创建的主管和员工。
  - 可以查看自己团队的所有订单，并审批下属的提现申请。
  - 支持配置支付网关（通过 `ENCRYPTION_KEY` 加密存入数据库）。
- **主管 (Supervisor)**
  - 团队中层，只能管理自己的直属下级员工。
  - 查看自己与下属的订单，发起自己的提现申请。
- **员工 (Employee / Staff)**
  - 基础执行者，无任何人员管理权限。
  - 只能管理自己名下的商品。
  - 查看自己的订单，发起自己的提现申请。

### 2.2 商品海报与支付流程
- **商品录入**: 上传图片，设定价格。
- **海报生成**: 使用 `canvas` 动态生成带有微信/支付宝付款二维码的营销海报（代码中实现了 `ctrip`、`meituan` 等多套海报模板）。
- **支付对账**: 在 `server.ts` 中暴露了 `/api/webhook/*` 接口用于接收第三方支付服务商的回调通知，回调后自动修改订单状态并向员工钱包累加余额。

### 2.3 钱包与提现审批
- **余额系统**: 员工个人的钱包余额根据其账号设置的 `earnings_rate`（收益比例，默认10%）从支付成功的订单中自动分成。
- **提现流程**: 
  - 员工发起申请（`status: pending`）。
  - 经理在后台【提现管理】中点击通过，系统会从钱包中扣除对应金额，并记录流水（`withdrawals` 表）。
