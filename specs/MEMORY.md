## 🛡️ 边界守卫规则（强制执行）

### 阶段隔离原则

| 阶段 | 触发条件 | 允许操作 | 禁止操作 |
|------|----------|----------|----------|
| **需求澄清** | 使用"需求澄清"类 Skill 或讨论需求 | 需求分析、提问、澄清 | ❌ 编写代码、技术方案、实现细节 |
| **技术设计** | 使用"技术设计"类 Skill 或讨论设计 | 技术方案、架构设计、接口定义 | ❌ 编写具体实现代码 |
| **编码实现** | 使用"编码实现"类 Skill 或明确要求写代码 | ✅ 编写代码 | - |

### 跨阶段保护
- 若用户在错误阶段询问实现细节，**必须**提醒：
  > "当前阶段是【需求/设计】，我们先完成这一步，再进入编码。"
- 然后引导用户继续当前阶段的工作

## 🎯 核心开发原则

### 1. 文档同步更新（强制）
每次代码修改**必须**同步更新以下文档：
- `specs/功能使用说明.md` - 功能说明文档
- `AGENTS.md` - 项目规范文件（如有架构变更）
- 相关代码注释

**要求**：
- 不能只改代码不改文档
- 文档要包含：前端组件、后端接口、数据库字段定义
- 改动要彻底，从头到尾完整实现

### 2. 完整性原则
改动一个功能时，必须完整处理：
1. **前端**：UI组件、状态管理、事件处理
2. **后端**：API接口、业务逻辑、错误处理
3. **数据库**：表结构、字段定义、索引
4. **文档**：使用说明、接口文档、变更记录

**禁止**：改一半留一半，导致功能不可用

### 3. 后端服务与数据库规范
每次涉及数据的功能，文档中必须包含：
- API 接口定义（路径、方法、参数、返回值）
- 数据库表结构和字段说明
- 接口调用示例

## 📝 文档编写规范

### 功能使用说明格式
```markdown
## 功能名称

### 前端组件
- 组件位置：`src/components/XXX.tsx`
- 状态管理：使用的 hooks 和状态
- 事件处理：用户交互逻辑

### 后端接口
| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/api/xxx` | POST | 创建xxx | 需登录 |

**请求参数**：
```json
{
  "field1": "string",  // 字段说明
  "field2": "number"   // 字段说明
}
```

**返回值**：
```json
{
  "success": true,
  "data": { ... }
}
```

### 数据库表定义
```sql
CREATE TABLE xxx (
  id UUID PRIMARY KEY,
  field1 VARCHAR(255),  -- 字段说明
  created_at TIMESTAMP
);
```

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| id | UUID | 主键 | auto |
```

## 🔧 项目特定规范

### 商品页助手项目

#### 用户管理体系
- **角色**：经理(manager) > 主管(supervisor) > 员工(staff)
- **账号创建**：员工账号由管理员统一创建，**前端不提供注册功能**
- **审核流程**：新用户需要管理员审核（如启用）

#### 支付配置
- 支付配置统一从环境变量读取
- 不再区分系统级和商户级配置

#### 技术栈
- 前端：React 19 + TypeScript + Vite + Tailwind CSS
- 后端：Express + TypeScript + tsx + PostgreSQL
- 认证：JWT + bcryptjs
