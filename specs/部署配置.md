# 部署配置指南

## 1. 自动化部署 (GitHub Actions)

本项目已经配置了完整的 `.github/workflows/deploy.yml` 自动化流水线。

### 1.1 触发条件
- **分支**: 仅在推送到 `main` 分支时触发自动部署。
- **目标服务器**: 线上服务器（如 Vultr VPS）。

### 1.2 需要在 GitHub 配置的 Secrets
进入您的 GitHub 仓库 -> **Settings** -> **Secrets and variables** -> **Actions**，添加以下环境变量：
- `HOST`: 服务器的公网 IP 地址。
- `USERNAME`: 登录用户名（例如 `root`）。
- `SSH_PRIVATE_KEY`: 服务器的私钥（或密码等认证方式）。

### 1.3 部署流程分析
1. GitHub 监听到 `main` 分支有新的 `push` 提交。
2. Actions 运行器自动连接至您的 VPS 服务器。
3. 在服务器上进入 `/root/payforme` 目录。
4. 执行 `git pull origin main` 拉取最新代码。
5. 自动清理并重新安装依赖 (`npm install`)。
6. 对于图像处理模块（Canvas / Sharp）执行 `npm rebuild canvas sharp --build-from-source` 以兼容系统环境。
7. 运行 `npm run build` 打包前端静态资源（Vite）。
8. 停止并重新启动 PM2 守护进程：`pm2 start server.ts --name payforme --interpreter ./node_modules/.bin/tsx`。

## 2. 生产环境配置 (.env)

您的服务器 `/root/payforme/.env` 文件**必须**包含以下参数：

```env
# 核心环境
NODE_ENV=production
PORT=5000

# 数据库 (PostgreSQL)
DATABASE_URL=postgresql://用户名:密码@localhost:5432/payforme

# 安全配置 (千万不要泄露)
JWT_SECRET=your_jwt_secret_key_here
ENCRYPTION_KEY=your_encryption_key_32_bytes_long_hex
```

## 3. 手动运维指令

如果您需要直接在服务器进行排查，请使用以下 PM2 命令：

- 查看运行状态: `pm2 list`
- 查看系统日志: `pm2 logs payforme` (或 `pm2 logs payforme --lines 50`)
- 重启服务: `pm2 restart payforme`
- 停止服务: `pm2 stop payforme`
- 监控 CPU/内存: `pm2 monit`

## 4. 前端静态文件托管
本项目的后端 `server.ts` 已经配置了 Express 静态文件中间件：
```typescript
app.use(express.static(path.join(__dirname, 'dist')));
```
因此，无需配置 Nginx 处理前端文件，只要确保 `npm run build` 成功生成了 `dist` 目录，通过 `http://服务器IP:5000` 即可直接访问前端页面。通常您只需要在服务器配置一层 Nginx 反向代理将 80/443 端口指向 5000 即可。
