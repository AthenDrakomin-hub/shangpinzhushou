## 核心文件说明

### 后端入口 `server.ts`

```
职责：Express 服务入口
内容：
├── 配置加载（环境变量、JWT、数据库）
├── 中间件配置（helmet、cors、compression）
├── 数据库初始化（建表、创建管理员）
├── API 路由定义
├── 静态文件服务
└── 服务启动
```

### 前端入口 `src/main.tsx`

```
职责：React 应用入口
内容：
├── React 根节点渲染
├── 全局样式导入
└── App 组件挂载
```

### 主应用组件 `src/App.tsx`

```
职责：前端主应用（单文件架构）
特点：
├── 所有页面组件内联
├── 路由通过状态管理
├── 约 6000+ 行代码
└── 包含所有业务逻辑
```

## 目录职责划分

### `/src/components/` - 可复用组件
| 组件 | 职责 |
|------|------|
| `LazyImage.tsx` | 图片懒加载，优化加载性能 |
| `SharePoster.tsx` | 海报生成与预览 |

### `/src/pages/` - 页面组件
| 页面 | 路由 | 职责 |
|------|------|------|
| `H5ProductPage.tsx` | `/h5/:id` | 买家下单页面 |
| `StaffWallet.tsx` | Dashboard 内嵌 | 员工钱包管理 |
| `PayConfigPage.tsx` | Dashboard 内嵌 | 支付配置 |
| `LoginPage.tsx` | `/` | 登录页面 |

### `/src/services/` - 业务服务层
| 服务 | 职责 |
|------|------|
| `authService.ts` | 登录、注册、Token 管理 |
| `dataService.ts` | 数据同步、订阅 |
| `productService.ts` | 商品 CRUD、支付配置 |
| `walletService.ts` | 钱包余额、收益计算 |
| `withdrawalService.ts` | 提现申请、审核 |
| `wechatPay.ts` | 微信支付接口封装 |

### `/src/utils/` - 工具函数
| 工具 | 职责 |
|------|------|
| `api.ts` | Fetch 封装、错误处理 |
| `crypto.ts` | 加密、解密工具 |
| `superpay.ts` | SuperPay SDK 封装 |

## 数据库表结构

### 核心表

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| `users` | 用户表 | id, email, role, status |
| `products` | 商品表 | id, user_id, name, price |
| `orders` | 订单表 | id, user_id, product_id, status |
| `wallets` | 钱包表 | id, user_id, balance |
| `withdrawals` | 提现表 | id, user_id, amount, status |

### 表关系

```
users (1) ──────< (n) products
  │
  ├──────< (n) orders
  │
  ├────── (1) wallets
  │
  └──────< (n) withdrawals
```

## 文件命名规范

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 组件文件 | PascalCase | `LoginPage.tsx` |
| 服务文件 | camelCase + Service | `authService.ts` |
| 工具文件 | camelCase | `api.ts` |
| 配置文件 | kebab-case | `vite.config.ts` |
| 样式文件 | 同组件名 | `index.css` |

## 注意事项

### 单文件架构限制
- `src/App.tsx` 包含所有页面逻辑，体积较大
- 后续重构建议：拆分为独立页面组件

### 静态资源
- 商品图片存储在 `uploads/products/`
- 构建后通过 Express 静态服务访问

### 环境配置
- `.env` 文件不提交到版本控制
- 使用 `.env.example` 作为配置模板
