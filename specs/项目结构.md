## 核心文件说明

### 后端入口 `server.ts`

```
职责：Express 服务入口、数据库交互与核心业务逻辑
内容：
├── 配置加载（环境变量读取、JWT配置、数据库配置）
├── 数据库初始化（建表、索引、创建默认管理员账号）
├── API 路由定义 (包含认证、商品、订单、支付回调、提现、统计等)
├── SuperPay 与九久支付的回调处理及验签逻辑
├── 静态文件服务 (图片上传与前端打包产物映射)
└── 服务启动
```

### 前端入口 `src/main.tsx`

```
职责：React 应用入口
内容：
├── React 根节点渲染
├── 全局样式导入
└── App 组件挂载
```

### 主应用组件 `src/App.tsx`

```
职责：前端主应用路由与状态控制
特点：
├── 维护当前的 View 状态 (currentView)
├── 根据 URL 进行特殊页面拦截 (如 H5 商品页、支付结果页)
├── 处理未登录与已登录状态下的页面分发
└── 包含全局的 Toast 提示和通用分享弹窗控制
```

## 目录职责划分

### `/src/components/` - 可复用组件
| 组件 | 职责 |
|------|------|
| `ShareProductModal.tsx` | 统一的商品海报生成与分享预览弹窗 |
| `ui/AppLayout.tsx` | 登录后的后台整体布局、侧边栏和响应式导航 |
| `ui/*` | 通用的基础 UI 组件 (Button, Input, Card, Modal 等) |

### `/src/pages/` - 页面组件
本系统前端已完成基于文件的组件化重构，主要页面如下：
| 页面 | 路由表现 | 职责 |
|------|------|------|
| `H5ProductPage.tsx` | `/h5/:id` | 用户扫码后看到的轻量级商品展示与支付选择页 |
| `LoginPage.tsx` | `/` | 员工/管理员统一登录入口 |
| `DashboardPage.tsx` | Dashboard 视图 | 数据统计大盘及快捷入口 |
| `ProductsPage.tsx` | Products 视图 | 商户商品列表管理 |
| `ProductCreatePage.tsx` | ProductCreate 视图 | 极简的商品创建页面 |
| `UserManagePage.tsx` | Employees 视图 | 员工管理，配置收益分成、修改密码、冻结等 |
| `SettingsPage.tsx` | Settings 视图 | 账户设置，以及经理专属的真实动态支付通道配置 |
| `StaffWallet.tsx` | Wallet 视图 | 员工专属的钱包与收益记录管理 |

### `/src/services/` - 业务服务层
| 服务 | 职责 |
|------|------|
| `authService.ts` | 封装了登录、注销等前端认证相关的操作及接口调用 |
| `wechatPay.ts` | 封装了微信支付（九久支付）的下单参数构造与回调验签算法 |

*(注：原有的部分 service 和 utils 如 db.ts、superpay.ts 等由于逻辑收敛到 server.ts 中，已被清理)*

## 数据库表结构

### 核心表

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| `users` | 用户/员工表 | id, email, role, status, earnings_rate, superpay_merchant_on 等 |
| `products` | 商品表 | id, user_id, name, price, status 等 |
| `orders` | 订单表 | id, user_id, product_id, amount, status, pay_type 等 |
| `wallets` | 钱包表 | id, user_id, balance, total_earnings 等 |
| `withdrawals` | 提现记录表 | id, user_id, amount, status, payment_method 等 |

### 表关系

```
users (1) ──────< (n) products
  │
  ├──────< (n) orders
  │
  ├────── (1) wallets
  │
  └──────< (n) withdrawals
```

## 注意事项

### 前后端分离与全栈部署
- 项目采用 React (Vite) + Express (Node.js) 架构。
- 生产环境下，通过 `npm run build` 打包前端，然后启动 `server.ts`，Node.js 将直接服务于 `/dist/index.html`，实现全栈一体化运行。

### 静态资源
- 商品图片上传后存储在后端的 `uploads/products/` 目录下。
- 前端通过静态资源路由 `/uploads/*` 进行访问。

### 环境变量与配置
- 数据库连接、监听端口等基础配置依赖 `.env` 环境变量。
- **支付配置** (支付宝、微信等通道密钥) 不再依赖 `.env`，而是由经理在后台“系统设置”中动态配置，保存在数据库的经理账号记录中，修改后实时生效。
