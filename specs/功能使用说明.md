## 2. 角色功能指南

### 2.1 首席工程师 (最高权限)

**核心特权**: 无视一切商户隔离限制，直接对底层数据库与系统全局参数进行维护。

1. **数据库管理** (`/database-admin`)
   - 左侧列表显示所有的底层数据表，点击表名即可查看最新的 50 条记录。
   - **SQL 执行器**: 提供直接对数据库执行 SQL 的能力。在文本框上方有“查数据、改角色、删记录、增删字段”等模板，点击即可填充，替换邮箱等参数后点击“执行 SQL”即可立即生效。

2. **系统管理** (`/settings`)
   - 可以在“全局系统配置”弹窗中，增删改查 `system_configs` 表的键值对配置，用于动态管理整个平台的系统标题、API 密钥等参数。

3. **全量员工管理** (`/users`)
   - 支持【列表视图】与【树状视图】切换。
   - 在树状图中，可直观地看到 `首席工程师 -> 经理 -> 主管 -> 员工` 的汇报关系。
   - **操作**: 将鼠标悬停在任何成员名字上，即可显示【添加下属】和【删除该成员】按钮，实现对全平台组织架构的指哪打哪。

### 2.2 经理与主管 (管理权限)

1. **员工管理**
   - 可以通过右上角“添加员工”按钮为团队创建新的下属账号。
   - 可以编辑下属员工的“提成比例 (Earnings Rate)”，该比例决定了下属商品售出后，分成的百分比。

2. **提现审批**
   - 进入“提现管理”模块，可以查看到团队成员发起的提现请求。
   - 核对线下转账后，点击“通过”或“拒绝”。通过后，员工钱包的冻结余额将被扣除。

3. **支付通道配置 (仅经理)**
   - 在“系统设置”页面，点击“支付通道配置”，录入第三方收款网关的私钥和 AppID（内容会被 `ENCRYPTION_KEY` 强加密存入底层数据库）。

### 2.3 员工 (基础业务)

1. **商品管理**
   - 点击“新建商品”，填写商品名称、价格、库存，上传封面图（后端使用 `multer` 和 `sharp` 处理图片）。
   - 商品创建后，可以生成并下载包含付款二维码的推广海报。

2. **订单与提成**
   - 当客户扫描您的商品海报付款后，订单会出现在“订单管理”中。
   - 系统会自动计算提成，并放入您的“我的钱包”中。

3. **发起提现**
   - 进入“我的钱包”，输入金额发起提现请求，等待上级审批即可。

## 3. 常见问题 (FAQ)

- **为什么我在前端修改了代码，服务器上没变化？**
  - 请确保代码已经通过 Git 推送到了 `main` 分支，此时 GitHub Actions 会自动触发服务器拉取、构建和 PM2 重启。如果您在本地测试，需执行 `npm run build`。如果在浏览器端，请使用 `Ctrl + F5` 强制刷新。

- **忘记密码怎么办？**
  - 在登录页点击“忘记密码”，输入您的账号绑定的“密保问题”和答案，验证通过后可重置密码。

- **为什么订单支付成功了，后台状态没变？**
  - 请检查您的回调地址（Webhook）是否配置正确，且服务器的 5000 端口（或 Nginx 代理端口）是否对公网开放。可以通过 PM2 日志 (`pm2 logs payforme`) 查看回调接口 `/api/webhook` 的报错情况。
