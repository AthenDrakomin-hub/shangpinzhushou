## 代码规范

### TypeScript 规范

#### 编码风格
- **遵循 Airbnb 编码规范**
- 所有文件使用 ES Module (`type: module`)
- 使用函数式组件 + Hooks

#### 类型定义
```typescript
// ✅ 推荐：显式类型定义
interface User {
  id: string;
  email: string;
  role: 'manager' | 'supervisor' | 'staff';
}

// ✅ 推荐：函数参数类型标注
function createUser(data: CreateUserDTO): Promise<User> { ... }

// ❌ 禁止：隐式 any
function processData(data) { ... }
```

#### 命名规范
| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 组件 | PascalCase | `LoginPage`, `ProductCard` |
| 函数 | camelCase | `handleLogin`, `fetchOrders` |
| 常量 | UPPER_SNAKE_CASE | `API_BASE_URL`, `MAX_RETRY` |
| 接口/类型 | PascalCase | `User`, `OrderStatus` |
| 文件 | 组件用 PascalCase，其他用 camelCase | `LoginPage.tsx`, `api.ts` |

### React 规范

#### 组件结构
```tsx
// ✅ 推荐的组件结构
const ProductCard: React.FC<ProductCardProps> = ({ product, onEdit }) => {
  // 1. Hooks 声明
  const [isExpanded, setIsExpanded] = useState(false);
  const queryClient = useQueryClient();
  
  // 2. 派生状态
  const discountedPrice = product.price * 0.9;
  
  // 3. 事件处理函数
  const handleClick = () => {
    onEdit(product.id);
  };
  
  // 4. 渲染
  return (
    <div className="product-card">
      {/* ... */}
    </div>
  );
};
```

#### Hooks 规范
```tsx
// ✅ 自定义 Hook 以 use 开头
function useProductList(userId: string) {
  const [products, setProducts] = useState<Product[]>([]);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    fetchProducts(userId).then(setProducts).finally(() => setLoading(false));
  }, [userId]);
  
  return { products, loading };
}
```

#### Hydration 错误预防
```tsx
// ❌ 禁止：在 JSX 中直接使用动态数据
<div>{Date.now()}</div>

// ✅ 正确：使用 useEffect + useState
const [timestamp, setTimestamp] = useState<number | null>(null);
useEffect(() => {
  setTimestamp(Date.now());
}, []);
return <div>{timestamp}</div>;
```

## API 规范

### RESTful 接口设计

#### URL 命名
```
✅ 推荐                    ❌ 禁止
/api/products              /api/getProducts
/api/products/:id          /api/product/:id
/api/orders                /api/order_list
```

#### HTTP 方法
| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 查询资源 | `GET /api/products` |
| POST | 创建资源 | `POST /api/products` |
| PUT | 更新资源 | `PUT /api/products/:id` |
| DELETE | 删除资源 | `DELETE /api/products/:id` |

#### 响应格式
```json
// ✅ 成功响应
{
  "success": true,
  "data": { ... }
}

// ✅ 错误响应
{
  "error": "错误信息"
}
```

#### 分页参数
```
GET /api/orders?page=1&limit=20&status=pending
```

## 数据库规范

### 表命名
- 使用复数形式：`users`, `products`, `orders`
- 小写下划线命名：`wallet_transactions`

### 字段命名
| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 主键 | `id` | `id UUID PRIMARY KEY` |
| 外键 | `{表名单数}_id` | `user_id`, `product_id` |
| 时间戳 | `{动作}_at` | `created_at`, `paid_at` |
| 布尔值 | `is_{状态}` | `is_active`, `is_verified` |

### 索引命名
```sql
CREATE INDEX idx_{表名}_{字段名} ON {表名}({字段名});
-- 示例
CREATE INDEX idx_orders_status ON orders(status);
```

## Git 规范

### 分支命名
```
main                    # 主分支
develop                 # 开发分支
feature/功能名称        # 功能分支
fix/问题描述            # 修复分支
```

### 提交信息
```
<type>(<scope>): <subject>

type:
  feat     新功能
  fix      修复 Bug
  docs     文档更新
  style    代码格式（不影响功能）
  refactor 重构
  test     测试
  chore    构建/工具

示例:
  feat(auth): 添加员工账号创建功能
  fix(order): 修复订单状态更新问题
  docs(api): 更新接口文档
```

## 文档规范

### 必须同步更新的文档
| 文档 | 更新时机 |
|------|----------|
| `specs/功能使用说明.md` | 功能变更、接口变更 |
| `AGENTS.md` | 架构变更、技术栈变更 |
| `MEMORY.md` | 新增开发习惯/规则 |
| `specs/*.md` | 产品规划、技术选型变更 |

### 文档内容要求
每次功能变更必须包含：
1. **前端**：组件位置、状态管理、事件处理
2. **后端**：API 接口定义、请求/响应格式
3. **数据库**：表结构、字段说明

## 开发流程规范

### 边界守卫规则
| 阶段 | 允许操作 | 禁止操作 |
|------|----------|----------|
| 需求澄清 | 需求分析、提问 | 写代码、技术方案 |
| 技术设计 | 技术方案、接口定义 | 写实现代码 |
| 编码实现 | 编写代码 | - |

### 完整性原则
改动功能时必须完整处理：
1. 前端 UI 组件
2. 后端 API 接口
3. 数据库表结构
4. 相关文档更新

**禁止**：改一半留一半，导致功能不可用

## 安全规范

### 密码处理
```typescript
// ✅ 使用 bcrypt 加密
const hashedPassword = await bcrypt.hash(password, 10);

// ✅ 比较密码
const isValid = await bcrypt.compare(password, hashedPassword);
```

### JWT 配置
```typescript
// ✅ 从环境变量读取密钥
const secret = process.env.JWT_SECRET;

// ❌ 禁止硬编码
const secret = 'my-secret-key';  // 危险！
```

### SQL 注入防护
```typescript
// ✅ 使用参数化查询
await pool.query('SELECT * FROM users WHERE id = $1', [userId]);

// ❌ 禁止字符串拼接
await pool.query(`SELECT * FROM users WHERE id = '${userId}'`);  // 危险！
```
