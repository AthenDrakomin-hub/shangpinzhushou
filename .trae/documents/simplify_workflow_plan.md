# 页面优化及工作流调整计划

## 1. 发现的问题总结 (Current State Analysis)

根据您的最新需求，当前的代码在以下方面与您期望的产品定位有偏差：

1.  **登录页面提示语冗余**：
    -   不需要出现“账号由管理员统一创建”的提示。
2.  **商品创建页面的配置冗余**：
    -   当前商品创建页面要求选择“商品分类”、“分享模板风格”和“支持的下单方式”。
    -   您希望这是一个轻量级的商品分享助手，不需要在创建时绑定分类和模板。
    -   下单方式不应该在商品级别配置，而是全局的：只要商户在全局配置了哪些支付方式，商品分享页就自动显示哪些支付方式。
3.  **商品列表及分享流程的繁琐**：
    -   现在的分享流程是在商品列表点击分享生成海报。您期望的是：创建商品后可以直接预览并生成海报。
    -   生成海报时才需要选择对应的视觉模板（如美团、京东等），并且支持保存相册和一键分享。
4.  **H5 支付页面的输入冗余**：
    -   当前的 H5 商品页面（即分享给用户的页面）要求填写买家的“姓名”和“电话”。
    -   既然是简单的分享助手，用户扫码后应该只看到商品信息并直接支付，不需要填写任何多余信息。

## 2. 修复计划 (Proposed Changes)

我将修改以下文件以匹配您的需求：

### 步骤 1: 登录页去除非必要提示
-   **文件**: `/workspace/src/pages/LoginPage.tsx`
-   **修改内容**: 移除底部关于“账号由管理员统一创建”的提示语句，保持页面干净。

### 步骤 2: 简化商品创建页面
-   **文件**: `/workspace/src/pages/ProductCreatePage.tsx`
-   **修改内容**:
    -   移除“商品分类”的选择组件。
    -   移除“分享模板风格”的选择组件。
    -   移除“支持的下单方式”的选择组件。
    -   在向后端提交创建请求时，只传递 `name`, `price`, `description`, `image` 等核心字段（如果后端 API 对其他字段有要求，可传空或默认值，不影响展示）。
    -   创建成功后，不再直接返回商品列表，而是自动弹出“预览和分享”弹窗，允许用户立即生成海报。

### 步骤 3: 优化 H5 商品展示与支付页
-   **文件**: `/workspace/src/pages/H5ProductPage.tsx`
-   **修改内容**:
    -   彻底移除“买家信息（姓名、电话）”的输入表单。
    -   在提交订单（`handlePayment`）时，`buyerName` 和 `buyerPhone` 传空字符串。
    -   保持原有的基于 URL `template` 参数渲染视觉样式的功能（这是扫码后用户看到的视觉效果来源）。
    -   支付方式的选择逻辑改为：从后端读取当前商户全局配置支持的支付渠道并展示，无需在商品级别控制。

### 步骤 4: 整合后端订单创建逻辑
-   **文件**: `/workspace/server.ts`
-   **修改内容**:
    -   确保 `/api/orders` 接口（创建订单）在没有买家姓名和电话时能正常工作。
    -   确认 `/api/poster/generate` 接口能正常处理带有 `template` 参数的海报生成。

## 3. 假设与决策 (Assumptions & Decisions)
-   **全局支付配置**：假设目前后端获取商户支付配置的逻辑已经存在（基于全局配置而不是商品特定配置），我将确保前端 H5 页面根据商户实际配置的支付网关（如是否配置了 SuperPay 和九久支付）来展示支付按钮。
-   **分享入口**：虽然商品创建后会直接弹出分享，但商品列表的分享按钮依然会保留，并使用相同的生成海报逻辑（选择模板 -> 生成 -> 预览保存）。
-   **模板参数传递**：海报上生成的二维码 URL 会携带商户在生成海报时选择的 `template` 参数（例如 `?template=meituan`），这样用户扫码打开的 H5 页面就会自动显示对应的品牌视觉。

## 4. 验证步骤 (Verification steps)
1.  进入登录页，检查底部提示语是否已清理。
2.  进入商品创建页，确认分类、模板和下单方式配置已移除。
3.  提交创建商品，确认成功后能直接调起分享/生成海报功能。
4.  在海报生成弹窗中选择不同的模板，确认海报正常生成。
5.  扫描（或访问）生成的 H5 链接，确认页面只显示商品信息和支付按钮，没有姓名和电话输入框，且能正常调起支付。
