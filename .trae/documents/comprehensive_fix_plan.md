# 项目深度修复与重构计划 (Plan)

根据您的详细反馈，我将对项目进行一次全面的深度修复和功能完善。以下是详细的实施计划：

## 一、 商品管理优化
1. **海报设计优化**: 
   - 修改 `posterService` 或海报生成组件，解决海报过于空旷、原价与现价重叠的问题。
   - 优化排版布局，使生成的图片更具营销美感。
2. **分享按钮无响应修复**: 
   - 修复返回列表后点击“分享”、“复制链接”等按钮无反应的 Bug，确保状态正确重置和事件绑定有效。
3. **商品删除与编辑**:
   - 修复删除商品显示成功但实际未删除的问题（排查后端 `DELETE /api/products/:id` 逻辑）。
   - **全面实现“编辑商品”功能**：点击编辑跳转到表单页，回显商品数据并支持修改保存。
4. **交互反馈**: 
   - 为各个关键按钮（分享、删除、编辑等）增加明确的操作提示和二次确认，防止员工误操作。

## 二、 订单管理白屏修复
- **修复 `r.filter is not a function`**: 
  - 这是因为前端直接将后端返回的 `{ orders: [], total: ... }` 对象赋给了数组状态。
  - 将 `OrdersPage.tsx` 中的状态赋值修正为 `setOrders(data.orders || [])`，彻底解决白屏崩溃。

## 三、 用户管理与三级分润系统 (核心)
1. **同级权限隔离**: 
   - 限制管理员/经理不能编辑其他同级经理的账号，主管不能编辑同级主管的账号。
2. **身份与归属标签**: 
   - 列表页为每位员工打上“上级归属”标签（如：`所属主管: 张三`），方便经理区分。
   - 支持**晋升机制**：上级可以在编辑时将员工提拔为“主管”。
3. **三级收益分配算法 (经理 -> 主管 -> 员工)**:
   - 彻底修复 `earnings_rate` 无法保存的后端 Bug。
   - 订单支付成功后，触发三级分润计算：
     - **常规情况**：主管比例70%，员工比例50%。订单100元 $\rightarrow$ 员工得 35元 (70%×50%)，主管得 35元 (70%-35)，经理得 30元 (100%-70%)。
     - **跳级情况**：主管比例为0%，员工比例20%。订单100元 $\rightarrow$ 员工绕过主管直接得 20元，主管 0元，经理 80元。
     - **直属情况**：经理直接创建的员工，员工按比例拿钱，剩下全归经理。
   - 确保经理、主管和员工都有独立的钱包，并同步更新资金。

## 四、 提现管理后端补齐
- **修复 404 报错**: 
  - 后端 `server.ts` 补齐缺失的商户提现管理路由：
    - `GET /api/merchant/withdrawals` (全部)
    - `GET /api/merchant/withdrawals/pending` (待审核)
    - `GET /api/merchant/withdrawals/stats` (数据统计)
    - `POST /api/merchant/withdraw/:id/approve` (通过)
    - `POST /api/merchant/withdraw/:id/reject` (拒绝)
    - `POST /api/merchant/withdraw/:id/pay` (代付)
- 废弃多余的旧提现路由。
