## 2. 钱包与利益分配链条验证
**现状分析**：
代码库已包含独立的 `wallets` 表，且与 `users` 表通过 `user_id` 一一对应。核心逻辑（`server.ts` 第 336-364 行）已经实现了**链式分润**：当底层员工产生收益时，系统会向上溯源（最多 3 层），按设定的 `earnings_rate` 逐级将分润金额存入主管和经理的独立钱包中。
**结论**：
经理和主管已有独立钱包，链条式利益分配机制已存在并能正常运转。

## 4. 首席工程师角色设计
### 4.1 基础权限设定
- **账号标识**：将 `athendrakomin@proton.me` 的 `role` 字段升级为 `chief_engineer`。
- **钱包隔离**：该角色主要负责维护平台，不参与分润，因此不为其分配/计算钱包。
- **权限控制**：在后端（`server.ts`）中增加 `chiefEngineerMiddleware` 中间件，允许该角色调用创建经理账号的接口，并在查询用户列表时能获取全量数据。

### 4.2 专属系统监控仪表盘 (Chief Engineer Dashboard)
开发一个仅限首席工程师访问的独立大盘，包含：
1. **项目运行大盘**：Node.js 运行环境、版本、运行时长 (Uptime)。
2. **资源监控**：CPU 使用率、系统内存占用情况 (Memory Usage)。
3. **数据库状态**：当前连接数、数据表容量概览。
4. **异常日志面板**：实时捕获并展示 `server.ts` 产生的错误日志 (Error Logs)。

### 4.3 用户管理树形视图 (Tree View)
- 修改现有的 `UserManagePage.tsx`，针对 `chief_engineer` 提供专属的**树形图 (Tree View) 展示模式**。
- 支持从首席工程师向下展开：`首席工程师 -> 经理 -> 主管 -> 员工`。
- 后端增加一个专门返回树状层级结构的接口（`/api/users/tree`）。
