
## [2.1.1] - 2026-04-12
### Fixed & Improved (Critical System Stability Update)
**此版本修复了系统底层的多个历史遗留问题，确保了项目后续部署和运行的绝对稳定。**

1. **PM2 部署重启崩溃问题（核心修复）**
   - **问题现象**：每次执行 `pm2 restart` 后，图片上传接口必定报错“上传出错，请稍后重试”，或者整个后端服务宕机（内存占用变为 `0b` 或一直卡在低内存）。
   - **根本原因**：之前使用 `pm2 start npm -- run start` 启动时，PM2 只能管理外层的 `npm` 包工头。重启时 PM2 杀死了 `npm`，但真正的 `node`（`tsx`）子进程变成了“僵尸进程”并持续占用 `5000` 端口。导致新启动的服务因端口冲突而瞬间崩溃。
   - **解决方案**：放弃了 npm 代理启动，改用最直接、标准的底层挂载方式：`pm2 start npx --name "payforme" -- tsx server.ts`。此后 PM2 直接管理 Node 进程，彻底杜绝了僵尸进程和端口冲突问题。

2. **前端数据“假缓存”问题（防缓存机制）**
   - **问题现象**：在手机浏览器或微信内编辑商品（例如打开“共享”开关）并保存后，重新打开弹窗或查看列表，状态依然显示为修改前（关闭状态）。
   - **根本原因**：部分移动端浏览器对 `fetch` 的 `GET` 请求执行了非常激进的本地缓存。当用户再次请求商品列表时，浏览器直接返回了缓存中的旧数据，并没有向服务器发起真实请求。
   - **解决方案**：在 `src/utils/apiClient.ts` 全局请求拦截器中，为所有 `GET` 请求自动追加了时间戳防缓存参数（`?_t=123456789`），并增加了 `Cache-Control: no-cache` 头部。确保了数据 100% 实时同步。

3. **商品共享（is_shared）功能补全**
   - **修复内容**：
     - 在后端启动文件（`server.ts`）中增加了针对旧数据库表的自动迁移脚本，自动为 `products` 表补全缺失的 `is_shared` 字段，解决了商品列表因字段缺失而白屏报错的问题。
     - 在前端的 `EditProductModal`（编辑弹窗）中补全了原本漏掉的“支持共享该商品”开关。
     - 修复了弹窗打开时未能正确读取商品当前 `is_shared` 状态导致开关重置的 Bug。
     - 为商品列表（`ProductsPage`）增加了醒目的绿色 `[共享]` 徽章，方便管理员识别。

4. **全局多语言（i18n）优化**
   - **修复内容**：将左侧全局导航菜单（系统监控、仪表盘、商品管理等）加入了国际化字典 `zh.json` 和 `en.json`。现在点击语言切换后，整个页面的菜单也会随之实时变为英文，解决了此前“切换语言没变化”的错觉。

5. **其他 UI 修复**
   - 修复了手机端商品详情页底部“立即支付”长按钮遮挡下方支付通道（如微信支付原生）的问题，增加了底部安全内边距（Safe Area）。
   - 移除了系统中重复多余的“深色模式”设置菜单（保留了顶部快捷月亮图标）。
